OpenMPT (1.13.03) Guide * Notes thinly disguised as a newbie tutorial

I've been using OpenMPT for some time, so I decided I'd make a short guide so that friends and strangers can learn how I go about my work process and read about what they need to know when exploring trackers.

The goal throughout this entry is to not use the plugins rack (but there's no shame in using it). For a list of plugins I use, scroll all the way down to this page.

What Should I Know Before I Start Composing?

Like all software, the toughest part will be getting used to the UI and figuring out where everything is. There are many other tracker softwares you can use, but OpenMPT/Modplug is the one I'm most familiar with.

In my opinion if you're just starting out, you'd probably get better outcomes creating electronic music over traditional music in trackers. While chiptune and jungle modules can have around 20 or even under 10 slots of audio, it's tough to make tracker jazz sound believable and "not sampled" unless you record the melodies yourself.

Why Should I Use Trackers?

It's not that music trackers offer more in terms of features. Not everyone HAS to use a music tracker, and they're even considered obsolete ware to some. However, there's a lot of workflow related differences between the conventional DAW and a music tracker.

  1. Keyboard based control.
    When working with trackers, you'll be using keybinds for the most part instead of dragging-dropping.
  2. Makes improvisation easier.
    What would otherwise be tedious on a conventional DAW, you can pitch, reverse, and stretch individual notes in the editor itself. Convenient if you want to create breakcore or fusion.
  3. Songwide overview
    The interface you see is what you'll see for the rest of the project. Instead of using a horizontal piano roll for each pattern, you input note commands on a vertical spread and can crosschannel samples.
  4. You can share modules with your friends :)
    Check out modarchive.org, there's a whole community around this type of thing that's great for scavenging samples or giving back.

These can either be advantages or drawbacks depending on what you need in order to compose.

How To Read Chart Notes

Open up the "Patterns" tab on a new module. The chart overview window might be a little daunting— it looks more like Microsoft Excel than it does something you'd write music in. Don't think too hard about it.

Each channel is separated by 4 columns: the "Note Column," the "Instrument Column," the "Volume Column," and the "Effect Column".

The note column defines which pitch the note is playing in. Not much to detail on here, but in order to switch octaves beyond what your computer keyboard can record, locate the swatch labeled "Octave" and you can click the arrows according to what scale you want to play in.

The instrument column determines which instrument/sample will play. Without it, you won't hear anything. For future reference instead of selecting instruments using the dropdown menu, you can highlight this column and type in an instrument-assigned number. You'd be doing a lot of typing and remembering, and can especially do the same for the volume and effects columns.

This column affects all that is volume related. By default, this column displays "v64" and 64 is the loudest value it can play. By double-clicking this note, you open "Note Properties" and can pick other transformations like panning and portamento. The effects column is empty by default, but you can choose over 30+ effects that the volume column doesn't house. I won't talk about this one because your own experimentation can only do justice for the effects column.

For a better indepth, read the official Manual: Patterns wiki entry.
